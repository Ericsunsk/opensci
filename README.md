# OPENSCI 评审工作台

一个面向科研项目评审与支持决策的单页工作台。它把高不确定性项目的处理过程收敛成四个可操作区块：

- `Inbox`
  管理活跃评审队列，按阶段、负责人、风险和下个决策节点处理案件。
- `Case Summary`
  展示提交快照、申请资源、进入本轮评审的原因，以及当前案件上下文。
- `Evidence & Review`
  保留证据来源链、补证请求、评审阶段与带上下文的评审动作。
- `Decision Packet`
  输出可提交委员会的支持方案，而不是停留在自由调参或抽象打分。

## 产品目标

这个产品聚焦的不是“找好项目”这类过宽的问题，而是一个更具体的评审问题：

`如何帮助评审者在证据不完整、验证周期长的情况下，对科研项目做出更克制、更可追踪的支持决策，同时让支持后的责任链和信任更新保持可见？`

它重点解决三件事：

1. 让评审者更快读懂一个案件，而不是先淹没在项目叙事里。
2. 让不确定性被转成有负责人和截止时间的评审工作。
3. 让最终输出变成一个可提交、可复核的决策包，而不是散落在页面各处的判断碎片。

## 当前功能

- 支持中 / 英双语界面切换
- 支持按阶段、负责人与风险压力筛选案件
- 为每条证据展示来源、提交人、验证人和最后检查时间
- 用统一时间线整合内部备注、系统动作与项目更新
- 用固定政策模板生成 `Decision Packet`
- 支持将决策包加入委员会队列，并复制决策备忘录

## 设计取向

这个版本刻意做了三项收敛：

- 去掉默认首页里的方法论解释层，优先进入工作区
- 去掉自由滑杆和开关式“调参台”，改为固定支持模板
- 去掉分散的日志模块，改为统一案件活动流
